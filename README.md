# Сеул · лендинг путешествия

Одностраничный сайт о поездке Сергея и Полины в Сеул (7–15 июня 2026):
интерактивная карта, расписание каждого дня по часам и краткая историческая справка.

Чистый статический проект — **сборка не нужна**. HTML, CSS и JavaScript без
фреймворков. Карта — [Leaflet](https://leafletjs.com/) + тайлы OpenStreetMap.

## Структура

Код разложен по слоям (подробно — в `PLAN.md`, раздел 2). Снаружи это один
статический сайт, внутри — чёткое разделение данных, хранилища и интерфейса,
чтобы позже подключить бэкенд (Supabase) и ИИ без переписывания.

```
seoul-trip/
├── index.html             — разметка страницы
├── PLAN.md                — план развития продукта и архитектура
├── css/
│   └── styles.css          — все стили
├── js/
│   ├── app.js              — точка входа: собирает слои воедино
│   ├── model/              — данные: сущности, генерация дней, seed Сеула
│   │   ├── entities.js     — формы данных (Trip, Place, Day, …) и фабрики
│   │   ├── days.js         — построение каркаса дней из дат
│   │   ├── config.js       — категории дней и их цвета
│   │   └── seed.seoul.js   — ГОТОВАЯ ПОЕЗДКА (редактируется здесь)
│   ├── data/               — хранилище за интерфейсом
│   │   ├── repository.js   — контракт доступа к данным
│   │   └── localStorageRepo.js — реализация на localStorage
│   ├── store/store.js      — состояние + уведомление интерфейса
│   ├── ai/                 — генерация маршрута за интерфейсом
│   │   ├── aiService.js    — контракт
│   │   └── mockAiService.js — заглушка (отдаёт места из seed)
│   └── ui/                 — интерфейс: карта, вкладки, расписание
├── package.json            — скрипт для запуска локального сервера
└── .gitignore
```

## Как запустить

**Вариант 1 — просто открыть файл.**
Дважды кликните `index.html` — страница откроется в браузере. Нужен только
интернет (для тайлов карты и библиотеки Leaflet с CDN).

**Вариант 2 — локальный сервер** (удобно при активной разработке):

```bash
npm run dev
```

Команда поднимет статический сервер (`serve`) и выдаст адрес вида
`http://localhost:3000`. Node.js нужен только для этого варианта.

## Как редактировать маршрут

Содержание готовой поездки лежит в `js/model/seed.seoul.js` — трогать остальной
код не нужно. (По мере развития места можно будет добавлять прямо из интерфейса.)

- **Добавить место** — добавьте объект в массив `places`:

  ```js
  { name:"Название", coords:[37.5, 127.0], dayNumber:3, time:"12:00",
    photo:"📍", price:2, by:"Сергей", source:"seed",
    desc:"Краткое описание.",
    history:"Историческая заметка (необязательно)." }
  ```

- **Поля места**
  - `dayNumber` — `0` отель, `1..7` дни прогулок, `8` вылет
  - `time` — время визита `'ЧЧ:ММ'` (расписание сортируется по нему)
  - `price` — `'free'`, `1`, `2` или `3` (ценовой сегмент)
  - `by` — `'Сергей'`, `'Полина'` или `'Оба'` (кто нашёл место)
  - `photo` — эмодзи; можно заменить на реальное фото (`<img>` в попапе)

- **Изменить день** — заголовки, даты и темы дней лежат в массиве `days`.
- **Перелёт и отель** — в полях `flights` и `hotel` той же поездки.
- **Категории дней** (Туризм, Районы, Шоппинг, Тренды) — в `js/model/config.js`.

## Технологии

- HTML + CSS + ванильный JavaScript, без сборки
- [Leaflet 1.9.4](https://leafletjs.com/) — карта
- Тайлы [OpenStreetMap](https://www.openstreetmap.org/) — без API-ключа
- Ссылки на места ведут в Kakao Map
